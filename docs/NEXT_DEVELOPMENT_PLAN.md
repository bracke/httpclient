# Next development plan after 1.0.0

This note is intentionally short. It prevents Phase 16 from becoming a feature-development phase.

Future work should begin only after the 1.0.0 release artifact and tag metadata are prepared. Candidate post-release tracks include compatibility-preserving fixes, documentation clarification, optional backend work for experimental HTTP/3, and additional interoperability fixtures. Any feature that changes public API shape, default security behavior, redirect/retry semantics, binary-body handling, or proxy/TLS routing must start with a separate compatibility review.
